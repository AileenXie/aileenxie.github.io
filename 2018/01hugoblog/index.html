Hugo快速搭建博客

Written by Aileen.Xie with ♥ on October 5, 2018 in 学习 · 笔记    本文总阅读量次

简单记录一下mac下用Hugo搭建博客的过程，以便日后查阅。感谢renyijiu的安利和帮助。

Hugo

  • 一个用Go语言写的静态网站生成器
  • 能把markdown转变成静态网页
  • 内置web服务期，便于本地草稿调试

安装 Hugo

  • 直接用Homebrew安装

    brew install hugo

  • 完成之后查看版本，我目前装的是0.49

    hugo version

    创建一个新站点

  • 到你的目录下找个喜欢的地方执行语句

    hugo new site myblog

会在当前目录生成一个叫“myblog”的目录，里面包含若干文件夹和一个config.toml文件:

▸ archetypes/

▸ content/ ->你写的markdown文章

▸ layouts/ ->网站的模板文件

▸ static/ ->放的是一些图片、css、js等资源

▸ data/

▸ themes/ ->放的是你之后添加的主题

config.toml ->网站的配置文件

添加主题

  • 进入目录页

    cd myblog

  • 初始化为git项目，方便之后放入github或者任何仓库进行版本管理

    git init

  • 添加主题, 以LeaveIt为例

    git submodule add  https://github.com/liuzc/LeaveIt.git themes/LeaveIt

    之后会在themes/目录下看到你添加的主题。

  • 修改配置文件(直接打开config.toml文件修改也是一样的)

    echo 'theme = "LeaveIt"' >> config.toml

  • Hugo中有很多可供选择的主题，选一个你喜欢的，复制链接替换上面的主题

添加第一篇博客

  • 在myblog目录下运行命令

    hugo new posts/my-first-post.md

  • 会在content/posts目录下生成一个my-first-post.md的文件，文件初始会有”title”(文章标题)、”date”（文章时间）、”draft”（是否是草稿）几个字段内容。之后你要写的内容以markdown写在后面即可，这里我们可以先不管。

  • 【tips】: 刚开始我在这步运行报错，查了资料发现是hugo版本问题，需要更新到0.48以上，如果装的版本旧了，手动更新一下：

    brew update
brew upgrade hugo

启动web服务器本地调试（看看效果）

  • 在myblog目录下运行命令

    hugo server

  • 访问：http://localhost:1313/ 可以看到博客的效果，值得一提的是，前面生成的.md文档里”draft”设置为”true”的，无法被看到。想要看到作为草稿的文章，记得加”-D”参数

    hugo server -D

调整你的主题

按照所用主题的例子修改你的config.toml文件.

  • 公共的部分：

    baseURL = "https://example.org/"
languageCode = "en-us"
title = "My New Hugo Site"
theme = "ananke"

  • 所用主题的定制部分参考主题介绍，如LeaveIt的config.toml中的头像配置 avatar = "/images/me/avatar.jpeg"把你想要的头像按照/images/meavatar.jpeg格式放到static目录下（注意不是项目根目录）

利用Hugo生成静态页面

  • 在myblog目录下运行命令

    hugo

  • 会自动生成/刷新public目录，之后每次修改过markdown文件或改过主题，都要重新运行hugo命令以更新public目录下的静态文件。

  • 【tips】:本地调试的时候不需要运行hugo,修改保存后会直接更新在页面上，但public下的静态文件是没有被更新的。

将public目录代码放到Github上

  • github上新建一个项目”blog”
  • 复制项目ssh链接地址: 如git@github.com:xxx/blog.git

  • 确保已上传ssh-key，进入本地myblog项目的public目录初始化git并添加远端仓库

    cd public
git init
git remote add origin git@github.com:xxx/blog.git

  • 提交你public下的代码

    git add .
git commit -m 'initial commit.'
git push origin master

  • 设置blog项目的”Repository name”为”xxx.github.io”(xxx是你的github用户名），通过github的page功能，你的博客就可以通过https://xxx.github.io/直接访问。（只提交public目录即可直接访问不用再做跳转。）

整个过程到此结束，之后的博客更新就是直接修改content/目录下的markdown文件，然后执行hugo后push更新到github。大功告成！
